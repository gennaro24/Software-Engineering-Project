Il metodo controlla se le stringhe relative al nome e cognome Contact sono valide per l'inserimento nella ContactList. La logica di validazione è la seguente:

  • Deve essere presente almeno un campo tra nome e cognome
  • Se un campo contiene spazi , " o ' viene considerato vuoto

◆ isValidEmail()

static boolean LowLevelClasses.ContactChecker.isValidEmail ( String e)
static

Il metodo controlla se una Stringa relativa alle email di un Contact è valida attraverso il matching di una regex. La logica di validazione è la seguente:

  • la prima parte della mail deve contenere almeno un carattere alfanumerico. sono consentiti i caratteri: (. _)
  • Deve contenere una sola chiocciola tra le due parti della mail.
  • la seconda parte della mail deve essere composta da una parte di mail agent separata da . dalla parte del dominio
  • la parte di domino deve contenere almeno due caratteri.
Parametri
euna Stringa che rappresenta la mail da validare.
Restituisce
un valore booleano che è true se la mail è valida, false altrimenti.

static

Il metodo controlla se una Stringa relativa ai numeri di un Contact è valida attraverso il matching di una regex. La logica di validazione è la seguente:

  • la stringa deve contenere solo numeri
  • deve avere una lunghezza compresa tra 3 e 15 caratteri.
Parametri
nuna Stringa che rappresenta il numero da validare.
Restituisce
un valore booleano che è true se la mail è valida, false altrimenti.
